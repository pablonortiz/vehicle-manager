# Project Instructions

## Workflow: Code Changes

After every code change (feature, bug fix, refactor, etc.):

1. **Test on device** using the `mcp-mobile-interaction` MCP on the Android emulator/device.
2. **If it works**: commit the change. Do NOT include self-credit in commit messages (no "by Claude", "AI-generated", etc.).
3. **If it doesn't work**: keep fixing and retesting until it passes, then commit.

Never commit untested code. Always verify visually on the device first.
